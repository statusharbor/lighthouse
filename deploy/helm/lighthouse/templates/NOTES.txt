{{- if not .Values.k8sstats.enabled }}

Cluster-shape metrics (k8s_node_count, k8s_pods_running,
k8s_node_cpu_used_percent, etc.) are NOT being collected — the
k8sstats collector needs a cluster-scoped ClusterRole that this
chart only installs when explicitly opted in:
  --set k8sstats.enabled=true

The added RBAC is: nodes/pods/PVCs list + nodes/proxy get (the
proxy verb is what reaches each kubelet's /stats/summary). Skip if
your security team audits cluster-scoped RBAC drift; the agent
still works without it — you just won't see the k8s_* metric
family on the Console.
{{- end }}

DaemonSet (per-node host metrics) is enabled. Each node now runs a

`restricted` Pod Security Admission profile forbids hostPath
volumes — label the namespace accordingly:
  kubectl label ns {{ .Release.Namespace }} pod-security.kubernetes.io/enforce=baseline
{{- end }}

  {{- if .Values.daemonset.enabled }}

  DaemonSet (daemonset.enabled=true):
    - LIGHTHOUSE_ROLE=host_metrics is unknown to old agents →
      DaemonSet pods run the check scheduler + discovery watcher
      (every check fires from every node — N× duplicate traffic
      against your targets).
    - LIGHTHOUSE_PROC_ROOT=/host/proc is silently ignored →
      reported CPU/mem/disk reflect the container's view, not the
      node's.
    - DaemonSet pods don't send runtime=kubernetes during register
      → allow_multi_instance never flips → pods 409 each other on
      the instance-id claim and CrashLoopBackOff.
  {{- end }}
  {{- if .Values.k8sstats.enabled }}

  k8sstats (k8sstats.enabled=true):
    - The cluster-shape collector lives in agent 1.12.0+. RBAC is
      installed but unused → no k8s_* metrics on the Console.
  {{- end }}

  Pin a 1.12.0+ image with:
    --set image.tag=1.12.0
{{- end }}

When you remove this Lighthouse from the Console, the agent exits and
the pod enters CrashLoopBackOff (k8s forces restartPolicy=Always on
StatefulSets, so the clean exit is treated as a crash). That's the
intended signal — your monitoring stack should pick it up. To remove
the pod cleanly:
  helm uninstall {{ .Release.Name }} -n {{ .Release.Namespace }}
